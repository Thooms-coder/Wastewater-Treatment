# Methods Log Workflow

Use this log as the running record of what was attempted, what failed, what changed, and what should happen next.

Committee guidance from the last progress meeting was explicit:

- document failed methods, not only successful ones
- treat the experimental outline as a working checklist
- keep writing and methods documentation moving in parallel with experiments

## Recommended usage

1. Copy `notes/methods_log_template.csv` to `notes/methods_log.csv`.
2. Add a new row every time a method, meter setup, chemistry condition, or analysis path changes.
3. Be specific in `what_failed` and `notes`.
4. Record the next action immediately so a stalled method has a clear follow-up.

## Suggested interpretation of `status`

- `Planned`: not yet run
- `Active`: currently being used
- `Blocked`: waiting on instrument, material, or site condition
- `Failed`: approach did not work and should not be retried without changes
- `Retired`: approach no longer in use

The point is not polished prose. The point is to preserve the experimental path so the thesis and publications do not lose failed attempts, pivots, and rationale.
