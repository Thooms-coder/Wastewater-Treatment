# Wastewater Odor Analytics Platform

## Overview

This repo is the current working research and reporting platform for wastewater odor analysis.

It combines:
- raw plant data currently available in `data/raw/`
- a preprocessing and feature-engineering pipeline for NH3, H2S, flow, events, and chemistry context
- a Streamlit dashboard for executive review, operations review, chemistry/dosing review, coverage, and diagnostics
- supporting research notes and figure-generation scripts

The `data/raw/` directory is the full set of raw source data currently available in this project as of now. There is no larger hidden raw dataset elsewhere in the repo.

## Current Architecture

### App layer

The Streamlit app is now split into focused modules:

- `app/app.py`: bootstrap only
- `app/app_state.py`: sidebar controls and filtered state assembly
- `app/data_services.py`: loading, event logic, aggregations, plotting wrappers
- `app/dashboard_ui.py`: shared UI, chemistry helpers, optional-table validation
- `app/page_renderers.py`: page-level rendering

### Pipeline layer

Core build and analysis scripts live in `scripts/`:

- `build_master.py`: loads raw data, preprocesses, engineers features, adds event and chemistry fields, writes `master_1min.parquet` and `master_1h.parquet`
- `build_daily.py`: aggregates minute data into daily summaries and coverage metrics
- `build_aggregates.py`: builds monthly and weekday summaries from daily data
- `build_modeling_table.py`, `train_models.py`: modeling path
- `analytics.py`, `plotting.py`: shared analysis and chart logic

### Research / notes

- `notes/`: research notes, design documents, and dissertation-alignment material

```bash
source venv/bin/activate
```

Windows:

```bash
python -m venv venv
venv\Scripts\activate
```

```bash

```

## Data

Expected folders:

```bash
mkdir -p data/raw data/processed
```

Current project assumption:

- `data/raw/` contains all raw source files currently available for this project
- `data/processed/` is generated by the pipeline

Optional processed CSV inputs used by the dashboard chemistry workflow:

- `data/processed/struvite_observations.csv`
- `data/processed/chemistry_lab_results.csv`

The dashboard now validates these optional tables more explicitly:

- `Struvite Observations` should include a date/timestamp field and a location-like field
- `Chemistry Lab Results` should include a date/timestamp field and chemistry result columns such as `pH`, `alkalinity`, `mg`, `nh4`, or `po4`

## Key Domain Concepts

### Signals

- `NH3` is treated as a more continuous signal and is summarized with a rolling mean
- `H2S` is treated as a spike-driven signal and is summarized with a rolling max

### Event detection

The app tracks ON/OFF transitions for:

- ferric chloride availability
- hydrochloric acid availability

### Event windows

- baseline: `-48h` to `-12h`
- post: `+12h` to `+96h`

### Chemistry support

The repo currently supports:

- ferric dose context
- HCl dose context
- mg/L conversion from active lb/day and flow
- optional struvite and lab-table ingestion

The repo does not yet provide full mechanistic chemistry modeling or full struvite outcome analysis.

## Tests

The repo now includes a runnable `unittest` suite.

Run all tests:

```bash
python -m unittest discover -s tests -v
```

Current test coverage includes:

- app state filtering and recomputation
- event and aggregation service logic
- chemistry mg/L conversions and optional-table validation
- fixture-driven processed-data loading
- end-to-end daily and aggregate build outputs
- page-renderer smoke coverage without browser automation
